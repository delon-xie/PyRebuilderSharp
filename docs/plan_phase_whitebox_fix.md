**版本**: v1.0

**前置**: Phase_log_fx · CrashCollector · `_processedBlockIds` · 孤儿块恢复

---

## 一、核心理念

**不再依赖矩阵测试的"通过率"。** 矩阵测试只验证反编译器不崩溃（形式正确），不验证输出是否与源代码语义一致（实质正确）。

白盒测试管道：逐个 .py 源文件 → 编译 → 反编译 → diff 对比 → 根因分析 → 修复 → 回归 → 归档。

---

## 二、测试流程（标准工作流）

```
┌────────────────────────────────┐
│ 1. 选择 test_data/input/*.py   │
│    记录文件大小、行数、Python 版本 │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 2. 使用 python3.10 编译为 .pyc │
│    python3.10 -c "py_compile"  │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 3. dotnet run (3.10) 反编译    │
│    输出到 /tmp/decompiled.py   │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 4. diff 对比                   │
│    diff <(反编译) <(源代码)     │
│    过滤注释/空行/版权头          │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 5. 对差异逐项进行根因分析       │
│    分类为:                      │
│    A) 预期差异 (注释丢失等)      │
│    B) 语义错误 (待修复)         │
│    C) CFG 限制 (已知问题)       │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 6. 修复 B 类问题               │
│    每修复一项 → dotnet build   │
│    → 回归 16 测试 + abc 快照   │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 7. 写 test_data/docs/           │
│    error_whitebox_fix_<name>.md │
│    记录问题、根因、修复          │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 8. 扩展到 2.7-3.14 多版本      │
│    重复上述步骤                 │
└────────┬───────────────────────┘
         ▼
┌────────────────────────────────┐
│ 9. 写 summary_whitebox_fix.md   │
│    归档代码                     │
└────────────────────────────────┘
```

## 三、验收标准

| 标准 | 说明 |
|:-----|:------|
| ✅ 逐块解析 | 每个基本块都进入 BuildStatements 或孤儿恢复 |
| ✅ 异常反馈 | 所有失败均生成 CommentBlock + CrashCollector 日志 |
| ✅ 不丢一块 | `_processedBlockIds` + `DecompiledInstructionOffsets` 覆盖全部 |
| ✅ 孤儿块注释化 | `# orphan @0x...` + 平坦反编译内容 |
| ✅ 回归稳定 | 16 关键测试 + abc 快照通过 |
| ⚠️ 语义一致 | diff 差异仅限 A 类（注释/空行/版权头丢失） |
| ❌ B/C 类记录 | 未修复的问题记入文档，不进代码级修复 |

## 五、已知限制（C 类）

1. CFG 边缺失 → try/except handler → class 定义（需 BlockScanner 修改）
2. `__doc__ = '...'` vs bare docstring → Python 编译器策略差异
3. 子控制流内的孤儿块 → 嵌套控制流重建不完整
4. 版权注释/空行丢失 → .pyc 不保存注释
