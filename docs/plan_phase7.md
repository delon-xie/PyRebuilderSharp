## 四、质量评价设计方案

### B1: 有源码质量评价

```
用户选择 .py 文件
  → 编译为 2.7, 3.5–3.14 各版本 .pyc
  → 每个版本独立反编译
  → AST 语义比较: original.py vs decompiled.py
  → 输出: 各版本还原度百分比 + 差异行
```

### B2: 无源码质量评价

```
用户选择 .pyc 文件
  → 反编译为 .py
  → 用同版本 Python 重新编译 .py → .pyc
  → dis 比较: original .pyc 的 dis vs recompiled .pyc 的 dis
  → 输出: 字节码还原度百分比
```

### B3: GUI 集成

```
Avalonia 面板:
  [选择源文件] [选择目标版本] [开始评价]
  ┌─────────────────────────────────────┐
  │ 质量报告                             │
  │ v3.12: 98.5% ✅  (12 diff lines)    │
  │ v3.11: 97.2% ✅  (15 diff lines)    │
  │ v3.10: 89.1% ⚠️  (48 diff lines)   │
  │ ...                                  │
  └─────────────────────────────────────┘
```
